# 文件结构定义

## 目标

本目录体系同时服务以下事情：

- 按阶段管理测试链路
- 按方案做横向对比
- 按轮次沉淀输出结果
- 从上游素材稳定生成基础提示词与节点提示词
- 后续新增方案时，先改本文件，再补齐真实目录

## 命名规则

- 文件夹优先使用中文，保留必要编号
- 阶段目录固定为 `01-第一阶段-线稿`、`02-第二阶段-上色`、`03-第三阶段-动作`
- 方案目录统一使用 `方案xx-名称`
- 轮次目录统一使用 `第01轮`、`第02轮`
- 节点提示词文件保留少量必要英文，统一使用 `node-xx-name.md`
- 测试记录文件统一使用中文语义名

## 上游公共素材链路

`00-公共资源/` 存放所有方案共用的输入素材与角色基础资料。

- `世界观设定/`：世界观设定、风格边界、题材约束
- `人物信息/`：每个角色的完整资料入口
- `原始素材/`：原图、截图、立绘、平视图、草稿图、补充资料
- `参考图/`：参考姿势、风格参考、构图参考

`人物信息/` 的下一层直接放具体人物目录，例如：

- `角色01-艾琳娜/`
- `角色02-凛/`
- `角色03-维安/`

每个角色目录统一包含：

- `参考图/`：角色灵感图、服装参考、发型参考、气质参考
- `角色介绍文档.md`：角色背景、性格、外观、服装等核心说明
- `补充说明/`：待确认点、人工判断、额外补充、派生说明

角色的基础提示词草稿不放在角色目录内部，统一整理到：

- `prompts/00-阶段0-原始元素处理/基础描述整理/`

根目录 `prompts/` 用于集中查看和维护实际测试中使用的提示词版本，按阶段分类，便于横向对比调整。

其中 `00-阶段0-原始元素处理/` 专门用于放测试开始前的预处理提示词，例如：

- 原始图清洗
- 角色元素提取
- 风格统一
- 基础描述整理

推荐工作顺序：

1. 在 `00-公共资源/` 中沉淀原始输入
2. 在根目录 `prompts/00-阶段0-原始元素处理/` 中整理预处理提示词
3. 在每个角色目录的 `角色介绍文档.md` 与 `补充说明/` 中整理角色信息
4. 在 `prompts/00-阶段0-原始元素处理/基础描述整理/` 中整理角色基础提示词草稿
5. 在根目录 `prompts/` 中按阶段和方案整理测试版提示词
6. 在 `节点截图/` 中保存节点截图、参数截图、流程截图
7. 在 `输出结果/第01轮/` 中保存首轮生成结果
8. 在 `测试记录/` 中记录问题、结论与下一轮调整方向

## 阶段与方案

### 01-第一阶段-线稿

第一阶段目标是得到高质量线稿，或得到可用于后续线稿流程的前置结果。

- `方案01-直接生成/`：直接从提示词生成前置图
- `方案02-骨架生成/`：先得到骨架或结构图
- `方案04-换头换衣服生成/`：基于已有角色特征做前置生成

### 02-第二阶段-上色

第二阶段目标是给第一阶段结果上色。

- `方案01-单人上色/`：单人上色方案
- `方案02-多人上色/`：多人上色预留方案

### 03-第三阶段-动作

第三阶段目标是根据设计图或前序结果生成动作表现。

- `方案01-单人动作/`：单人动作生成

## 统一子结构模板

每个方案目录都保持同一结构：

- `提示词/`
- `节点截图/`
- `输出结果/第01轮/`
- `测试记录/`

根目录 `prompts/` 推荐结构与阶段目录保持一致：

- `prompts/00-阶段0-原始元素处理/原始图清洗/`
- `prompts/00-阶段0-原始元素处理/角色元素提取/`
- `prompts/00-阶段0-原始元素处理/风格统一/`
- `prompts/00-阶段0-原始元素处理/基础描述整理/`
- `prompts/01-第一阶段-线稿/方案01-直接生成/`
- `prompts/01-第一阶段-线稿/方案02-骨架生成/`
- `prompts/01-第一阶段-线稿/方案04-换头换衣服生成/`
- `prompts/02-第二阶段-上色/方案01-单人上色/`
- `prompts/02-第二阶段-上色/方案02-多人上色/`
- `prompts/03-第三阶段-动作/方案01-单人动作/`

推荐文件模板：

- `提示词/node-01-input.md`
- `提示词/node-02-controlnet.md`
- `提示词/node-03-refiner.md`
- `测试记录/第01轮-测试记录.md`
- `测试记录/第01轮-问题总结.md`
- `测试记录/第01轮-阶段结论.md`

## 图片命名规则

所有复制或移动进仓库的图片，都要按经过的流程方案重命名，避免后期无法判断来源。

推荐格式：

- `角色名-阶段名-方案名-轮次-序号.png`
- `角色名-第一阶段-骨架生成-第01轮-01.png`
- `角色名-第二阶段-单人上色-第01轮-02.png`

如果同一张图经过多个流程节点，可以继续追加关键信息：

- `角色名-第一阶段-换头换衣服生成-controlnet-第01轮-01.png`

## 新增方案流程

1. 先更新本文件中的阶段说明与方案列表
2. 新建对应 `方案xx-*` 目录
3. 补齐 `提示词/`、`节点截图/`、`输出结果/第01轮/`、`测试记录/`
4. 复制模板文件并改为该方案专用内容
5. 开始新一轮测试
